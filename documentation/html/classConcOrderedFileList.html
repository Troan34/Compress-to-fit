Detailed Description

template<SerializableToDisk T>
class ConcOrderedFileList< T >

First sequence in first sequence out, i.e. the data is written in the order given to insert().

Template Parameters
TMust implement write_to(ofstream&) or be trivially copyable

Implemented as doubly linked list. A separate write thread is spawned.

Note
Because the list will own the elements, consider using a memory pool (e.g. pmr) as to not reallocate memory.

Inserts an element to the list, which it will own. THE FIRST ELEMENT MUST HAVE SEQUENCE NUMBER == 0, check warning for additional info.

Parameters
[in]elementElement to be inserted
sequence_numThe number of this element's sequence
Precondition
element must not own a nullptr
Postcondition
Any reference to element is invalidated.
Warning
THE FIRST ELEMENT THAT IS GOING TO BE WRITTEN TO FILE MUST HAVE SEQUENCE NUMBER == 0. This requirement exists as the first element you add could easily be not the first data chunk (i.e. out of order). Failure to follow this requirement will cause DEADLOCK.
